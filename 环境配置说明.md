# Spring Boot 环境配置完整说明

## 一、什么是环境配置？

**环境配置** 是指为不同的运行环境（开发、测试、生产）准备不同的配置参数。这样可以：
- 在开发环境使用本地数据库
- 在测试环境使用测试数据库
- 在生产环境使用正式数据库
- 避免敏感信息泄露

## 二、配置文件的层级关系

### 1. 主配置文件：`application.yml`
这是**基础配置文件**，包含所有环境通用的配置。

```yaml
spring:
  profiles:
    active: dev  # 激活哪个环境配置
```

### 2. 环境特定配置文件
- `application-dev.yml`：开发环境配置
- `application-test.yml`：测试环境配置
- `application-prod.yml`：生产环境配置

**Spring Boot 的加载顺序：**
1. 先加载 `application.yml`
2. 再加载 `application-{active}.yml`（根据 `spring.profiles.active` 的值）
3. 环境特定配置会**覆盖**基础配置中的相同项

## 三、`${}` 占位符详解

### 1. `${}` 的作用
`${变量名}` 是 Spring 的**属性占位符**，用于引用其他地方定义的配置值。

### 2. `${}` 里面的名称从哪来？

#### 方式一：引用同一配置文件中的其他属性

**在 `application.yml` 中：**
```yaml
spring:
  datasource:
    driver-class-name: ${sky.datasource.driver-class-name}
    url: jdbc:mysql://${sky.datasource.host}:${sky.datasource.port}/${sky.datasource.database}
    username: ${sky.datasource.username}
    password: ${sky.datasource.password}
```

**在 `application-dev.yml` 中定义具体值：**
```yaml
sky:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    host: localhost
    port: 3306
    database: sky_take_out
    username: root
    password: 123456
```

**解释：**
- `${sky.datasource.host}` 会被替换为 `localhost`
- `${sky.datasource.port}` 会被替换为 `3306`
- 这样写的好处：主配置文件不需要改，只需要修改环境特定文件

#### 方式二：引用环境变量

```yaml
sky:
  datasource:
    username: ${DB_USERNAME:root}        # 从环境变量 DB_USERNAME 读取，默认值为 root
    password: ${DB_PASSWORD}             # 从环境变量 DB_PASSWORD 读取，无默认值
  alioss:
    access-key: ${OSS_ACCESS_KEY_ID}     # 从环境变量读取
    secret-key: ${OSS_ACCESS_KEY_SECRET} # 从环境变量读取
```

**语法说明：**
- `${变量名}`：读取变量，如果不存在会报错
- `${变量名:默认值}`：读取变量，如果不存在使用默认值

#### 方式三：引用 Java 系统属性

```yaml
app:
  java-home: ${java.home}      # 读取 Java 系统属性
  user-home: ${user.home}      # 读取用户主目录
```

## 四、密码的正确配置方式

### ⚠️ 错误方式（不安全）

```yaml
# application-dev.yml - 直接写密码（不推荐）
sky:
  datasource:
    password: 123456  # ❌ 密码明文暴露在代码中
```

### ✅ 正确方式一：使用环境变量（推荐）

**1. 在配置文件中使用占位符：**

```yaml
# application-dev.yml
sky:
  datasource:
    username: ${DB_USERNAME:root}      # 开发环境可以有默认值
    password: ${DB_PASSWORD:123456}    # 开发环境可以有默认值
```

```yaml
# application-prod.yml
sky:
  datasource:
    username: ${DB_USERNAME}           # 生产环境不设默认值，强制从环境变量读取
    password: ${DB_PASSWORD}           # 生产环境不设默认值，强制从环境变量读取
```

**2. 在服务器上设置环境变量：**

**Linux/Mac：**
```bash
# 临时设置（当前会话有效）
export DB_USERNAME=root
export DB_PASSWORD=your_secure_password

# 永久设置（添加到 ~/.bashrc 或 ~/.bash_profile）
echo 'export DB_USERNAME=root' >> ~/.bashrc
echo 'export DB_PASSWORD=your_secure_password' >> ~/.bashrc
source ~/.bashrc
```

**Windows：**
```cmd
# 临时设置
set DB_USERNAME=root
set DB_PASSWORD=your_secure_password

# 永久设置（系统环境变量）
setx DB_USERNAME "root"
setx DB_PASSWORD "your_secure_password"
```

**IDEA 运行配置：**
1. 打开 Run → Edit Configurations
2. 选择你的 Spring Boot 应用
3. 在 Environment variables 中添加：
   ```
   DB_USERNAME=root;DB_PASSWORD=123456;OSS_ACCESS_KEY_ID=your_key;OSS_ACCESS_KEY_SECRET=your_secret
   ```

### ✅ 正确方式二：使用独立的本地配置文件

**1. 创建本地配置文件（不提交到 Git）：**

```yaml
# application-dev-local.yml（添加到 .gitignore）
sky:
  datasource:
    password: 123456
  alioss:
    access-key: LTAI5tYourAccessKey
    secret-key: YourSecretKeyHere
```

**2. 激活多个配置文件：**

```yaml
# application.yml
spring:
  profiles:
    active: dev,dev-local  # 同时激活 dev 和 dev-local
```

**3. 在 `.gitignore` 中添加：**
```
application-dev-local.yml
application-*-local.yml
*.local.yml
```

### ✅ 正确方式三：使用配置中心

在企业级应用中，使用配置中心（如 Nacos、Apollo、Spring Cloud Config）统一管理配置。

## 五、阿里云 OSS 配置完整示例

### 1. 修改 `AliOssProperties.java` 支持 access-key 和 secret-key

```java
@ConfigurationProperties(prefix = "sky.alioss")
@Data
public class AliOssProperties {
    private String endpoint;
    private String bucketName;
    private String regionId;
    private String accessKey;    // 新增
    private String secretKey;    // 新增
}
```

### 2. 开发环境配置（使用环境变量）

```yaml
# application-dev.yml
sky:
  alioss:
    endpoint: oss-cn-beijing.aliyuncs.com
    region-id: cn-beijing
    bucket-name: sky-takeout-dev           # 开发环境用的 bucket
    access-key: ${OSS_ACCESS_KEY_ID}       # 从环境变量读取
    secret-key: ${OSS_ACCESS_KEY_SECRET}   # 从环境变量读取
```

### 3. 生产环境配置

```yaml
# application-prod.yml
sky:
  alioss:
    endpoint: oss-cn-shanghai.aliyuncs.com
    region-id: cn-shanghai
    bucket-name: sky-takeout-prod          # 生产环境用的 bucket
    access-key: ${OSS_ACCESS_KEY_ID}       # 必须从环境变量读取
    secret-key: ${OSS_ACCESS_KEY_SECRET}   # 必须从环境变量读取
```

### 4. 设置环境变量

```bash
# 开发环境
export OSS_ACCESS_KEY_ID=LTAI5tYourDevAccessKey
```

## 七、如何配置环境？

### 方式一：修改 application.yml

```yaml
spring:
  profiles:
    active: dev  # 改为 prod 即切换到生产环境
```

### 方式二：启动时指定（推荐生产环境）

```bash
# 命令行参数
java -jar sky-server.jar --spring.profiles.active=prod

# 环境变量
export SPRING_PROFILES_ACTIVE=prod
java -jar sky-server.jar

# JVM 参数
java -Dspring.profiles.active=prod -jar sky-server.jar
```

### 方式三：在 IDEA 中配置

1. Run → Edit Configurations
2. 选择 Spring Boot 应用
3. VM options 中添加：`-Dspring.profiles.active=dev`
4. 或在 Environment variables 中添加：`SPRING_PROFILES_ACTIVE=dev`

## 八、安全最佳实践

### ✅ 应该做的
1. **敏感信息使用环境变量**：密码、密钥、token 等
2. **不同环境使用不同配置文件**：开发/测试/生产分离
3. **本地配置文件添加到 .gitignore**：避免提交敏感信息
4. **生产环境配置不设默认值**：强制通过环境变量提供
5. **定期轮换密钥**：定期更换数据库密码和 API 密钥

### ❌ 不应该做的
1. ❌ 在代码或配置文件中硬编码密码
2. ❌ 将包含真实密码的配置文件提交到 Git
3. ❌ 在日志中打印敏感信息
4. ❌ 所有环境使用相同的密码和密钥
5. ❌ 将生产环境密钥放在开发机器上

## 九、常见问题解答

### Q1: `${sky.datasource.password}` 这个名字是怎么来的？
**A:** 这是自定义的属性路径，对应 YAML 中的层级结构：
```yaml
sky:              # 第一层
  datasource:     # 第二层
    password: xxx # 第三层
```
完整路径就是 `sky.datasource.password`

### Q2: 为什么有些配置在 application.yml，有些在 application-dev.yml？
**A:**
- **application.yml**：放通用配置和占位符引用
- **application-dev.yml**：放开发环境的具体值
- 这样切换环境时只需要改 `spring.profiles.active`，无需改代码

### Q3: 密码到底应该写在哪里？
**A:**
- **开发环境**：可以写在 `application-dev.yml` 或使用环境变量
- **生产环境**：必须使用环境变量或密钥管理服务，绝不能硬编码

### Q4: 如何验证配置是否生效？
**A:**
1. 启动应用时查看日志，会显示激活的 profile
2. 在代码中注入配置类并打印（不要打印密码）
3. 使用 Spring Boot Actuator 的 `/actuator/env` 端点查看
