# 环境变量配置指南

## 问题说明

本项目使用阿里云OSS进行文件存储，需要配置环境变量来提供访问凭证。如果环境变量未正确配置，会导致以下错误：

```
com.aliyun.oss.OSSException: You have no right to access this object because of bucket acl.
[ErrorCode]: AccessDenied
```

## 为什么使用环境变量？

1. **安全性**：避免将敏感信息（如AccessKey）直接写在配置文件中
2. **灵活性**：不同环境（开发、测试、生产）可以使用不同的配置
3. **最佳实践**：符合12-Factor应用的配置管理原则

## 所需环境变量

| 变量名 | 说明 | 是否必需 |
|--------|------|----------|
| `OSS_ACCESS_KEY_ID` | 阿里云OSS访问密钥ID | 是 |
| `OSS_ACCESS_KEY_SECRET` | 阿里云OSS访问密钥Secret | 是 |
| `DB_USERNAME` | 数据库用户名 | 否（默认：root） |
| `DB_PASSWORD` | 数据库密码 | 否（默认：123456） |

## 配置方法

### 方法一：在 IDEA 中配置（推荐用于开发）

1. 打开 IDEA，点击右上角的运行配置下拉菜单
2. 选择 "Edit Configurations..."
3. 在左侧选择你的 Spring Boot 启动配置（如 `SkyApplication`）
4. 在右侧找到 "Environment variables" 字段
5. 点击文本框右侧的文件夹图标或直接输入
6. 添加环境变量（格式：`KEY=VALUE`，多个变量用分号分隔）：
   ```
   OSS_ACCESS_KEY_ID=你的AccessKeyId;OSS_ACCESS_KEY_SECRET=你的AccessKeySecret
   ```
7. 点击 "Apply" 和 "OK" 保存配置

**图示说明：**
```
Run/Debug Configurations
├── Name: SkyApplication
├── Main class: com.sky.SkyApplication
├── Environment variables: [点击此处]
│   ├── OSS_ACCESS_KEY_ID=LTAI5t...
│   └── OSS_ACCESS_KEY_SECRET=abc123...
└── [Apply] [OK]
```

### 方法二：使用 .env 文件（推荐用于团队协作）

1. 复制项目根目录下的 `.env.example` 文件为 `.env`：
   ```bash
   cp .env.example .env
   ```

2. 编辑 `.env` 文件，填入真实的密钥：
   ```properties
   OSS_ACCESS_KEY_ID=你的AccessKeyId
   OSS_ACCESS_KEY_SECRET=你的AccessKeySecret
   ```

3. 在 IDEA 中安装 EnvFile 插件：
   - File → Settings → Plugins
   - 搜索 "EnvFile" 并安装
   - 重启 IDEA

4. 在运行配置中启用 .env 文件：
   - Edit Configurations
   - 找到 "EnvFile" 标签页
   - 点击 "+" 添加 `.env` 文件
   - 勾选 "Enable EnvFile"

### 方法三：系统环境变量（推荐用于生产环境）

#### Windows 系统

**永久配置（推荐）：**

1. 右键 "此电脑" → "属性"
2. 点击 "高级系统设置"
3. 点击 "环境变量"
4. 在 "用户变量" 或 "系统变量" 中点击 "新建"
5. 添加以下变量：
   - 变量名：`OSS_ACCESS_KEY_ID`
   - 变量值：你的AccessKeyId
   - 变量名：`OSS_ACCESS_KEY_SECRET`
   - 变量值：你的AccessKeySecret
6. 点击 "确定" 保存
7. **重启 IDEA 或命令行窗口**以使环境变量生效

**临时配置（仅当前命令行窗口有效）：**
```cmd
set OSS_ACCESS_KEY_ID=你的AccessKeyId
set OSS_ACCESS_KEY_SECRET=你的AccessKeySecret
```

**注意**：使用 `set` 命令设置的环境变量只在当前命令行窗口有效，关闭窗口后失效。如果你在命令行中用 `set` 设置后再用 IDEA 运行，IDEA 无法读取这些变量。

#### Linux / macOS 系统

**临时配置（当前终端会话）：**
```bash
export OSS_ACCESS_KEY_ID=你的AccessKeyId
export OSS_ACCESS_KEY_SECRET=你的AccessKeySecret
```

**永久配置：**

编辑 `~/.bashrc` 或 `~/.zshrc` 文件，添加：
```bash
export OSS_ACCESS_KEY_ID=你的AccessKeyId
export OSS_ACCESS_KEY_SECRET=你的AccessKeySecret
```

然后执行：
```bash
source ~/.bashrc  # 或 source ~/.zshrc
```

### 方法四：Maven 启动时指定（用于命令行运行）

```bash
mvn spring-boot:run -Dspring-boot.run.arguments="--OSS_ACCESS_KEY_ID=你的AccessKeyId --OSS_ACCESS_KEY_SECRET=你的AccessKeySecret"
```

或者使用系统属性：
```bash
export OSS_ACCESS_KEY_ID=你的AccessKeyId
export OSS_ACCESS_KEY_SECRET=你的AccessKeySecret
mvn spring-boot:run
```

### 方法五：JAR 包启动时指定

```bash
java -jar sky-server.jar --OSS_ACCESS_KEY_ID=你的AccessKeyId --OSS_ACCESS_KEY_SECRET=你的AccessKeySecret
```

或者使用环境变量：
```bash
export OSS_ACCESS_KEY_ID=你的AccessKeyId
export OSS_ACCESS_KEY_SECRET=你的AccessKeySecret
java -jar sky-server.jar
```

## 验证配置是否生效

启动应用后，查看日志。如果环境变量未配置，会在上传文件时报错：
```
com.aliyun.oss.OSSException: You have no right to access this object because of bucket acl.
[ErrorCode]: AccessDenied
```

如果配置正确，文件上传应该能够成功。

## 获取阿里云 OSS 密钥

1. 登录[阿里云控制台](https://home.console.aliyun.com/)
2. 点击右上角头像，选择 "AccessKey 管理"
3. 创建 AccessKey，会生成 AccessKey ID 和 AccessKey Secret
4. **重要**：AccessKey Secret 只在创建时显示一次，请妥善保管

## OSS Bucket 权限配置

如果环境变量配置正确但仍然报 AccessDenied 错误，需要检查 OSS Bucket 的权限：

1. 登录[阿里云 OSS 控制台](https://oss.console.aliyun.com/)
2. 找到你的 Bucket（如 `sky-takeout`）
3. 检查以下配置：
   - **Bucket ACL**：建议设置为 "私有" 或 "公共读"
   - **跨域设置（CORS）**：如果需要浏览器直传，需要配置 CORS 规则
   - **访问控制**：确保你的 AccessKey 有读写权限

4. 检查 RAM 用户权限（如果使用 RAM 子账号）：
   - 在 RAM 控制台为用户添加 `AliyunOSSFullAccess` 策略
   - 或者添加自定义策略允许对特定 Bucket 的操作

## 常见问题

### 1. 为什么在命令行用 `set` 设置后 IDEA 还是读不到？

**原因**：`set` 命令只在当前命令行窗口设置环境变量，IDEA 是独立进程，无法读取。

**解决**：
- 方法一：在 IDEA 的运行配置中直接设置环境变量
- 方法二：在系统环境变量中永久设置
- 方法三：使用 .env 文件 + EnvFile 插件

### 2. 配置了环境变量但还是报 AccessDenied？

可能的原因：
1. **AccessKey 没有权限**：检查 RAM 控制台，为用户授予 OSS 权限
2. **Bucket 不存在或名称错误**：检查 `application-dev.yml` 中的 `bucket-name`
3. **Region 配置错误**：确保 `endpoint` 和 `region` 匹配
   - 例如：endpoint 是 `oss-cn-beijing.aliyuncs.com`，region 应该是 `cn-beijing`
4. **Bucket ACL 设置过于严格**：在 OSS 控制台调整 Bucket 的访问权限

### 3. 如何验证环境变量是否设置成功？

**Windows：**
```cmd
echo %OSS_ACCESS_KEY_ID%
echo %OSS_ACCESS_KEY_SECRET%
```

**Linux/macOS：**
```bash
echo $OSS_ACCESS_KEY_ID
echo $OSS_ACCESS_KEY_SECRET
```

**在 Java 代码中：**
```java
System.out.println("OSS_ACCESS_KEY_ID: " + System.getenv("OSS_ACCESS_KEY_ID"));
```

### 4. 生产环境如何配置？

生产环境建议使用：
1. **容器化部署**（Docker/Kubernetes）：在容器配置中注入环境变量
2. **云服务器**：在服务器系统环境变量中配置
3. **配置中心**：使用 Spring Cloud Config、Nacos 等配置中心管理
4. **密钥管理服务**：使用阿里云 KMS、AWS Secrets Manager 等

## 安全建议

1. **不要提交密钥到代码仓库**：将 `.env` 文件添加到 `.gitignore`
2. **定期轮换密钥**：建议每 90 天更换一次 AccessKey
3. **使用 RAM 子账号**：不要使用主账号的 AccessKey，创建 RAM 子账号并授予最小权限
4. **启用 MFA**：为阿里云账号启用多因素认证
5. **监控访问日志**：定期检查 OSS 访问日志，发现异常访问及时处理
