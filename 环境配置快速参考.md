# 环境配置快速参考

## 一、${} 占位符语法速查

### 基本用法
```yaml
# 从其他配置读取
database:
  url: ${app.db.host}

# 从环境变量读取
database:
  password: ${DB_PASSWORD}

# 带默认值（如果环境变量不存在，使用默认值）
database:
  username: ${DB_USERNAME:root}
  password: ${DB_PASSWORD:123456}
  port: ${DB_PORT:3306}
```

### 占位符来源优先级
1. Java 系统属性（-D 参数）
2. 环境变量
3. 配置文件中定义的值
4. 默认值（: 后面的值）

## 二、密码配置的三种正确方式

### 方式 1：环境变量（推荐生产环境）
```yaml
# application-prod.yml
sky:
  datasource:
    password: ${DB_PASSWORD}  # 必须设置环境变量
```

**设置环境变量：**
```bash
export DB_PASSWORD=your_secure_password
```

### 方式 2：环境变量 + 默认值（适合开发环境）
```yaml
# application-dev.yml
sky:
  datasource:
    password: ${DB_PASSWORD:123456}  # 没有环境变量时使用 123456
```

### 方式 3：本地配置文件（不提交到 Git）
```yaml
# application-dev-local.yml（添加到 .gitignore）
sky:
  datasource:
    password: 123456
```

**激活本地配置：**
```yaml
# application.yml
spring:
  profiles:
    active: dev,dev-local  # 逗号分隔多个 profile
```

## 三、阿里云 OSS 配置示例

### 完整配置
```yaml
sky:
  alioss:
    endpoint: oss-cn-beijing.aliyuncs.com
    region-id: cn-beijing
    bucket-name: sky-takeout
    access-key: ${OSS_ACCESS_KEY_ID}      # 从环境变量读取
    secret-key: ${OSS_ACCESS_KEY_SECRET}  # 从环境变量读取
```

### 设置环境变量
```bash
# Linux/Mac
export OSS_ACCESS_KEY_ID=LTAI5tYourAccessKey
export OSS_ACCESS_KEY_SECRET=YourSecretKeyHere

# Windows CMD
set OSS_ACCESS_KEY_ID=LTAI5tYourAccessKey
set OSS_ACCESS_KEY_SECRET=YourSecretKeyHere

# Windows PowerShell
$env:OSS_ACCESS_KEY_ID="LTAI5tYourAccessKey"
$env:OSS_ACCESS_KEY_SECRET="YourSecretKeyHere"
```

### IDEA 环境变量配置
1. Run → Edit Configurations
2. 选择 Spring Boot 应用
3. Environment variables 中添加：
   ```
   DB_USERNAME=root;DB_PASSWORD=123456;OSS_ACCESS_KEY_ID=your_key;OSS_ACCESS_KEY_SECRET=your_secret
   ```

## 四、切换环境的方法

### 方法 1：修改 application.yml
```yaml
spring:
  profiles:
    active: dev  # 改为 prod、test 等
```

### 方法 2：启动时指定（推荐）
```bash
# 方式 A：命令行参数
java -jar app.jar --spring.profiles.active=prod

# 方式 B：JVM 参数
java -Dspring.profiles.active=prod -jar app.jar

# 方式 C：环境变量
export SPRING_PROFILES_ACTIVE=prod
java -jar app.jar
```

## 五、配置文件说明

| 文件名 | 用途 | 是否提交到 Git |
|--------|------|---------------|
| application.yml | 主配置，所有环境通用 | ✅ 提交 |
| application-dev.yml | 开发环境配置 | ✅ 提交（不含密码） |
| application-prod.yml | 生产环境配置 | ✅ 提交（不含密码） |
| application-dev-local.yml | 本地开发密码等 | ❌ 不提交 |
| .env | 环境变量配置 | ❌ 不提交 |
| .env.example | 环境变量示例 | ✅ 提交 |

## 六、常见问题

### Q: 为什么 ${sky.datasource.password} 找不到值？
A: 确保在 application-dev.yml 中定义了 `sky.datasource.password`，或者设置了对应的环境变量。

### Q: 配置了环境变量但不生效？
A:
1. 检查变量名是否正确（区分大小写）
2. 重启 IDE 或终端
3. 在 IDEA 中检查 Run Configuration 的环境变量设置

### Q: 如何验证配置是否正确？
A:
1. 启动应用，查看日志中的 profile 信息
2. 在代码中注入配置类并打印（不要打印密码）
3. 使用 @Value 注解读取配置值进行测试

### Q: 生产环境应该怎么配置？
A:
1. 所有密码和密钥通过环境变量提供
2. 不在配置文件中设置默认值
3. 使用配置中心（Nacos、Apollo）统一管理
4. 定期轮换密码和密钥
