# 环境变量配置问题修复说明

## 问题描述

您遇到的错误如下：

```
com.aliyun.oss.OSSException: You have no right to access this object because of bucket acl.
[ErrorCode]: AccessDenied
```

## 问题根源分析

### 1. PowerShell 命令错误

您在 PowerShell 中使用了错误的命令：

```powershell
# ❌ 错误的方式 - set 是 CMD 命令，不是 PowerShell 命令
set DB_PASSWORD=123456
set OSS_ACCESS_KEY_ID=LTAI5tYourKey
set OSS_ACCESS_KEY_SECRET=你的密钥
```

**问题**: PowerShell 不认识 `set` 命令！`set` 是 CMD (命令提示符) 的命令，在 PowerShell 中不起作用。

### 2. 正确的 PowerShell 设置方法

```powershell
# ✅ 正确的方式 - 使用 $env: 语法
$env:DB_PASSWORD = "123456"
$env:OSS_ACCESS_KEY_ID = "LTAI5tYourKey"
$env:OSS_ACCESS_KEY_SECRET = "你的AccessKeySecret"

# 验证是否设置成功
echo $env:OSS_ACCESS_KEY_ID
echo $env:OSS_ACCESS_KEY_SECRET
```

### 3. 为什么会出现 AccessDenied 错误？

因为环境变量没有正确设置，所以：
1. 应用无法读取到 `OSS_ACCESS_KEY_ID` 和 `OSS_ACCESS_KEY_SECRET`
2. 阿里云 OSS SDK 使用了无效或空的凭证
3. OSS 服务器拒绝了访问请求，返回 `AccessDenied` 错误

## 推荐的解决方案

### 方案一：在配置文件中直接设置（推荐用于开发环境）

编辑 `sky-server/src/main/resources/application-dev.yml`：

```yaml
sky:
  datasource:
    username: root
    password: 123456
  alioss:
    endpoint: oss-cn-beijing.aliyuncs.com
    region: cn-beijing
    access-key: LTAI5tYourAccessKeyID      # 直接设置你的 AccessKeyID
    secret-key: 你的AccessKeySecret         # 直接设置你的 AccessKeySecret
    bucket-name: sky-takeout
```

**优点**：
- ✅ 不需要设置环境变量
- ✅ 配置简单，直接在配置文件中修改
- ✅ 适合本地开发环境

**缺点**：
- ⚠️ 不要将包含真实密钥的配置文件提交到 Git 仓库
- ⚠️ 不适合生产环境

### 方案二：在 IntelliJ IDEA 中设置环境变量（推荐）

1. 打开 **Run → Edit Configurations**
2. 选择 `SkyApplication` 运行配置
3. 找到 **Environment variables** 字段
4. 点击文件夹图标，添加环境变量：
   ```
   OSS_ACCESS_KEY_ID=LTAI5tYourAccessKeyID
   OSS_ACCESS_KEY_SECRET=你的AccessKeySecret
   DB_PASSWORD=123456
   ```
5. 点击 **OK** 保存
6. 重新运行应用

**优点**：
- ✅ 凭证不会被提交到 Git 仓库
- ✅ 只在 IDE 运行时有效，安全性更高
- ✅ 适合本地开发环境

### 方案三：在 PowerShell 中正确设置环境变量

```powershell
# 设置环境变量（正确的 PowerShell 语法）
$env:OSS_ACCESS_KEY_ID = "LTAI5tYourAccessKeyID"
$env:OSS_ACCESS_KEY_SECRET = "你的AccessKeySecret"
$env:DB_PASSWORD = "123456"

# 验证设置
echo $env:OSS_ACCESS_KEY_ID

# 然后在同一个 PowerShell 窗口中运行应用
cd D:\soft\idea_2023.3.7\data\sky-take-out
mvn spring-boot:run
```

**优点**：
- ✅ 凭证不会被提交到 Git 仓库
- ✅ 符合生产环境的最佳实践

**缺点**：
- ⚠️ 关闭 PowerShell 窗口后需要重新设置
- ⚠️ 在 IDE 中运行时，IDE 可能读不到这些变量（除非 IDE 是从该 PowerShell 窗口启动的）

## 如何验证配置是否正确

### 1. 启动应用后查看日志

正确配置后，应该看到以下日志之一：

```
INFO com.sky.utils.AliyunOSSOperator : 使用配置文件中的OSS凭证
```

或

```
INFO com.sky.utils.AliyunOSSOperator : 配置文件中未找到OSS凭证，尝试从环境变量读取
```

### 2. 测试文件上传

启动应用后，尝试上传文件。如果配置正确，应该看到：

```
INFO com.sky.utils.AliyunOSSOperator : 上传文件到OSS: bucket=sky-takeout, objectName=2026/03/xxx.jpg
INFO com.sky.utils.AliyunOSSOperator : 文件上传成功
```

### 3. 如果仍然出现错误

新的代码会提供更详细的错误信息：

```
OSS凭证未配置！请设置以下环境变量：
  - OSS_ACCESS_KEY_ID (或 ALIBABA_CLOUD_ACCESS_KEY_ID)
  - OSS_ACCESS_KEY_SECRET (或 ALIBABA_CLOUD_ACCESS_KEY_SECRET)
或者在application-dev.yml中配置：
  sky.alioss.access-key
  sky.alioss.secret-key
详细配置方法请参考 ENVIRONMENT_SETUP.md 文件
```

根据错误提示进行相应的配置即可。

## 获取阿里云 OSS 凭证

如果您还没有阿里云 OSS 凭证：

1. 登录阿里云控制台: https://www.aliyun.com
2. 进入 **访问控制 (RAM)** → **用户** → **创建用户**
3. 勾选 **OpenAPI调用访问**
4. 保存生成的 **AccessKey ID** 和 **AccessKey Secret**
5. 为用户授予 **AliyunOSSFullAccess** 权限（或根据需要授予最小权限）

## 常见错误对照表

| 错误现象 | 原因 | 解决方法 |
|---------|------|---------|
| 在 PowerShell 中 `set` 命令不识别 | 使用了 CMD 命令 | 改用 `$env:变量名 = "值"` |
| AccessDenied 错误 | OSS 凭证无效或未设置 | 检查凭证是否正确设置 |
| 环境变量设置后无效 | IDE 启动时还没有设置环境变量 | 重启 IDE 或在 IDE 中配置环境变量 |
| Bucket 不存在错误 | Bucket 名称错误或不存在 | 检查 bucket-name 配置 |
| 网络连接错误 | endpoint 或 region 配置错误 | 检查 endpoint 和 region 配置 |

## 安全建议

1. ✅ **不要将真实的 AccessKey 提交到 Git 仓库**
   - 将 `application-dev.yml` 添加到 `.gitignore` 中
   - 或者只提交模板文件 `application-dev.yml.template`

2. ✅ **使用环境变量存储敏感信息**
   - 在开发环境中使用 IDE 的环境变量配置
   - 在生产环境中使用系统环境变量或密钥管理服务

3. ✅ **使用 RAM 用户而不是主账号**
   - 创建专用的 RAM 用户用于应用访问
   - 只授予必要的权限（最小权限原则）

4. ✅ **定期轮换密钥**
   - 定期更换 AccessKey
   - 如果密钥泄露，立即禁用并重新生成
